# دليل مشروع متجر Larry Soft (Flutter & Sqflite)

هذا المشروع عبارة عن تطبيق متجر متكامل تم بناؤه باستخدام إطار العمل **Flutter** وقاعدة البيانات المحلية **Sqflite**، مع مراعاة كافة المتطلبات الأكاديمية والتقنية المذكورة في المحاضرات المرفوعة.

## 1. المميزات الرئيسية للتطبيق

| الميزة                | الوصف                                                                 |
| :------------------- | :-------------------------------------------------------------------- |
| **نظام المصادقة**    | شاشة تسجيل دخول (Login) وتسجيل حساب جديد (Register) مرتبطة بقاعدة البيانات. |
| **حفظ الحساب**       | خيار "تذكرني" (Remember Me) لحفظ بيانات الدخول باستخدام `SharedPreferences`. |
| **عرض المنتجات**     | واجهة رئيسية تعرض المنتجات في شبكة جذابة مع معلومات السعر والوصف.      |
| **سلة التسوق**       | إمكانية إضافة المنتجات، حذفها، وحساب المجموع الكلي بشكل آلي.           |
| **إدارة المظهر**     | دعم الوضع المظلم (Dark Mode) والوضع الفاتح مع حفظ الخيار المفضل.        |
| **قاعدة بيانات محلية**| استخدام `sqflite` لتخزين المستخدمين، المنتجات، وعناصر السلة بشكل دائم. |

## 2. هيكل الكود البرمجي (Files Structure)

تم تنظيم الكود ليكون سهل الفهم والشرح للدكتور:

- **`lib/main.dart`**: نقطة البداية، تهيئة التطبيق، وربط نظام الثيمات.
- **`lib/models/models.dart`**: يحتوي على نماذج البيانات (User, Product, CartItem).
- **`lib/services/database_helper.dart`**: القلب النابض للتطبيق، يدير جميع عمليات قاعدة البيانات.
- **`lib/providers/app_provider.dart`**: يدير "حالة" التطبيق (تسجيل الدخول، السلة، الثيم) ويضمن تحديث الواجهات فورياً.
- **`lib/screens/`**: يحتوي على جميع شاشات التطبيق (Login, Register, Home, Cart, Settings).

## 3. كيفية تشغيل المشروع

1. تأكد من تثبيت **Flutter SDK** على جهازك.
2. قم بفك ضغط ملف المشروع.
3. افتح المجلد في محرر الكود (مثل VS Code أو Android Studio).
4. قم بتشغيل الأمر `flutter pub get` في التيرمينال لتحميل التبعيات.
5. أضف الصور المطلوبة في مجلد `assets/images/` كما هو موضح في دليل الصور المرفق.
6. قم بتشغيل التطبيق على محاكي (Emulator) أو جهاز حقيقي.

## 4. تعليمات هامة للدكتور

عند تقديم المشروع، يمكنك التركيز على النقاط التالية التي تبرز جودة العمل:

- **التعامل غير المتزامن (Asynchronous Programming):** استخدام `async` و `await` في جميع عمليات قاعدة البيانات لضمان عدم تجمد الواجهة.
- **إدارة الحالة (State Management):** استخدام نمط `Provider` لفصل منطق الأعمال (Business Logic) عن واجهات المستخدم (UI).
- **تصميم قاعدة البيانات:** الجداول منظمة مع وجود مفاتيح أجنبية (Foreign Keys) لربط المنتجات بالسلة والمستخدمين.
- **تجربة المستخدم (UX):** دعم الوضع المظلم وحفظ الإعدادات يعطي انطباعاً احترافياً للتطبيق.

## 5. دليل الصور

يرجى مراجعة ملف `image_instructions.md` المرفق لمعرفة كيفية تسمية وإضافة الصور الخاصة بك لكي تظهر في التطبيق.
